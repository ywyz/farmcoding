## 1. 游戏概述

《The Farmer Was Replaced》是一款编程驱动的农场自动化游戏。玩家通过编写Python-like代码控制无人机，实现农作物的自动种植、收获、资源管理和农场扩张。游戏采用非标准Python环境，具有独特的语法限制和内置函数集。

### 1.1 基本玩法

- 使用无人机在网格状农场上进行自动化操作
- 种植不同作物获取资源
- 通过资源解锁新功能和升级
- 优化代码效率以提升产出速度

### 1.2 坐标系统

- 农场为方形网格，默认大小可通过 `get_world_size()` 获取
- X轴：从西(West)到东(East)，起点为0
- Y轴：从南(South)到北(North)，起点为0
- 超出边界会自动循环（从另一侧出现）

## 2. 核心机制系统

### 2.1 农场管理系统

#### 2.1.1 地块类型

| 地块类型 | 描述 | 功能 |
|---------|------|------|
| `Grounds.Grassland` | 草地 | 默认地块，草会自动生长 |
| `Grounds.Soil` | 土壤 | 通过 `till()` 翻耕得到，用于种植作物 |

#### 2.1.2 地块操作

- `till()`：将草地翻耕为土壤，或将土壤恢复为草地
- 种植作物需要土壤（部分作物可在草地上生长）

### 2.2 作物种植与收获系统

#### 2.2.1 作物类型

| 作物 | 生长时间 | 生长地块 | 产出资源 | 特殊机制 |
|-----|---------|---------|---------|---------|
| `Entities.Grass` | 0.5秒 | 草地/土壤 | `Items.Hay` | 自动在草地上生长 |
| `Entities.Bush` | 4秒 | 草地/土壤 | `Items.Wood` | 基础木材来源 |
| `Entities.Tree` | 7秒 | 草地/土壤 | `Items.Wood` | 产出更多木材，相邻树木会互相减速 |
| `Entities.Carrot` | 6秒 | 土壤 | `Items.Carrot` | 需要胡萝卜种子 |
| `Entities.Pumpkin` | 2秒 | 土壤 | `Items.Pumpkin` | 合并机制，20%枯萎率 |
| `Entities.Sunflower` | 5秒 | 土壤 | `Items.Power` | 花瓣数量机制 |
| `Entities.Cactus` | 1秒 | 土壤 | `Items.Cactus` | 排序连锁收获 |
| `Entities.Bush` | - | 草地/土壤 | - | 可转化为迷宫 |

#### 2.2.2 基础种植流程

```python
# 翻耕土地
till()

# 检查资源并种植
if num_items(Items.Carrot) > 0:
    plant(Entities.Carrot)
```

#### 2.2.3 收获机制

- `can_harvest()`：检查作物是否成熟
- `harvest()`：收获当前地块作物
- 枯萎南瓜 `Entities.Dead_Pumpkin` 上 `can_harvest()` 始终返回 `False`

### 2.3 水分管理系统

#### 2.3.1 水分机制

- 地块含水量范围：0 ~ 1
- 生长速度：含水量0时为1倍速，含水量1时为5倍速（线性增长）
- 蒸发：每秒平均失去当前含水量的1%
- 维持高含水量消耗更多水

#### 2.3.2 浇水操作

- 每10秒自动获得1罐水（升级 `Unlocks.Watering` 后翻倍）
- 1罐水增加0.25含水量
- 使用 `use_item(Items.Water)` 浇水

### 2.4 肥料与感染系统

#### 2.4.1 肥料效果

- 每10秒自动获得1份肥料（升级后翻倍）
- `use_item(Items.Fertilizer)`：缩短植物剩余生长时间2秒
- 副作用：施肥生长的植物会被感染

#### 2.4.2 感染机制

- 被感染植物收获时，一半产量变为 `Items.Weird_Substance`
- `use_item(Items.Weird_Substance)`：切换植物及相邻植物的感染状态
- 可用于治愈感染植物或感染健康植物

### 2.5 伴生种植系统（Polyculture）

#### 2.5.1 机制说明

- 草、灌木、树、胡萝卜有伴生偏好
- 通过 `get_companion()` 获取当前地块植物的伴生需求
- 返回值格式：`(plant_type, (x, y))` 或 `None`
- 在指定位置种植对应伴生植物后收获，产量增加

#### 2.5.2 产量加成

- 解锁前：5倍产量
- 每次升级：额外增加3倍

#### 2.5.3 使用示例

```python
companion = get_companion()
if companion != None:
    plant_type, (x, y) = companion
    # 移动到伴生位置并种植
    move_to(x, y)
    plant(plant_type)
```

### 2.6 向日葵能量系统

#### 2.6.1 能量机制

- 收获向日葵获得 `Items.Power`（能量）
- 能量自动使用，无人机以两倍速度运行
- 每30次行动消耗1点能量

#### 2.6.2 花瓣加成机制

- 花瓣数量范围：7 ~ 15
- 农场上至少10株向日葵时，收获花瓣最多的向日葵可获得8倍能量
- 通过 `measure()` 测量花瓣数量（成熟前也可测量）
- 多株向日葵可能有相同最大花瓣数

#### 2.6.3 策略要点

- 优先种植足够数量的向日葵（>=10株）
- 追踪各向日葵花瓣数量
- 优先收获花瓣最多的向日葵以获取8倍加成

### 2.7 南瓜合并机制

#### 2.7.1 合并规则

- 方形区域内所有南瓜完全成熟时合并为巨型南瓜
- 枯萎南瓜阻止合并
- 收获巨型南瓜获得产量 = 合并数量³（n*n*n）
- 当n >= 6时，产量 = n*n*6

#### 2.7.2 枯萎机制

- 南瓜完全成熟后有20%概率枯萎
- 枯萎南瓜变为 `Entities.Dead_Pumpkin`
- 枯萎南瓜不产出任何资源
- 在枯萎位置上种植新植物会自动移除枯萎南瓜

#### 2.7.3 最优策略

- 最佳种植大小：6x6（产量最大化）
- 及时补种枯萎南瓜
- 使用肥料加速生长，注意治愈感染

### 2.8 仙人掌连锁收获系统

#### 2.8.1 排序规则

仙人掌处于已排序状态的条件：
- 北边和东边相邻仙人掌：完全成熟且尺寸 >= 自身
- 南边和西边相邻仙人掌：完全成熟且尺寸 <= 自身

#### 2.8.2 连锁收获

- 收获已排序的仙人掌会递归收获所有相邻已排序仙人掌
- 产量 = 收获数量的平方（n²）
- 未排序仙人掌显示为棕色，排序后变回绿色

#### 2.8.3 仙人掌大小

- 大小范围：0 ~ 9
- 通过 `measure()` 获取当前地块仙人掌大小
- 通过 `measure(direction)` 获取指定方向相邻地块仙人掌大小
- 使用 `swap(direction)` 与相邻仙人掌交换位置

### 2.9 迷宫系统

#### 2.9.1 迷宫生成

- 在灌木上使用 `use_item(Items.Weird_Substance, amount)` 生成迷宫
- 迷宫大小取决于使用数量
- 未升级时：n份奇异物质生成nxn迷宫
- 每次升级使宝藏金币翻倍，但所需奇异物质也翻倍

#### 2.9.2 迷宫探索

- 无人机无法穿过树篱 `Entities.Hedge`
- 宝藏位置可通过 `measure()` 从迷宫任意位置获取
- 返回值为宝藏坐标 `(x, y)`
- `get_entity_type()` 在宝藏位置返回 `Entities.Treasure`

#### 2.9.3 宝藏收获

- 对宝藏使用 `harvest()` 获得等于迷宫面积的金币
- 在其他地方使用 `harvest()` 会使迷宫消失
- 可通过 `can_move()` 或 `move()` 检查墙壁

#### 2.9.4 迷宫重复使用

- 在宝藏上再次使用相同数量奇异物质可移动宝藏
- 每次移动可能随机移除部分墙壁（产生循环）
- 宝藏最多可重新定位300次
- 金币不会增加，难度会提升

### 2.10 解锁与升级系统

#### 2.10.1 解锁类型

| 解锁 | 功能 |
|-----|------|
| `Unlocks.Expand` | 扩展农场 |
| `Unlocks.Plant` | 种植功能 |
| `Unlocks.Carrots` | 胡萝卜种植 |
| `Unlocks.Pumpkins` | 南瓜种植 |
| `Unlocks.Sunflowers` | 向日葵种植 |
| `Unlocks.Cactus` | 仙人掌种植 |
| `Unlocks.Polyculture` | 伴生种植 |
| `Unlocks.Mazes` | 迷宫系统 |
| `Unlocks.Watering` | 浇水系统 |
| `Unlocks.Fertilizer` | 肥料系统 |
| `Unlocks.Trees` | 树木种植 |
| `Unlocks.Speed` | 无人机速度 |
| `Unlocks.Senses` | 感知功能 |
| `Unlocks.Loops` | 循环功能 |
| `Unlocks.Functions` | 自定义函数 |
| `Unlocks.Lists` | 列表使用 |
| `Unlocks.Dictionaries` | 字典和集合 |
| `Unlocks.Operators` | 运算符 |
| `Unlocks.Variables` | 变量赋值 |

#### 2.10.2 解锁检查

```python
# 检查是否已解锁
if num_unlocked(Unlocks.Carrots) > 0:
    plant(Entities.Carrot)

# 获取解锁成本
cost = get_cost(Unlocks.Carrots)

# 执行解锁
unlock(Unlocks.Carrots)
```

### 2.11 多无人机系统（Megafarm）

#### 2.11.1 无人机管理

- `spawn_drone(task, *args)`：生成新无人机执行任务
- `wait_for(drone)`：等待指定无人机完成
- `has_finished(drone)`：检查无人机是否完成
- `num_drones()`：获取当前无人机数量
- `max_drones()`：获取最大无人机数量

#### 2.11.2 使用示例

```python
def harvest_column():
    for _ in range(get_world_size()):
        harvest()
        move(North)

# 生成新无人机
if num_drones() < max_drones():
    drone = spawn_drone(harvest_column)
```

## 3. 内置函数参考

### 3.1 作物管理函数

#### harvest()

| 属性 | 说明 |
|-----|------|
| **功能** | 收获当前地块的作物 |
| **参数** | 无 |
| **返回值** | `bool` - 是否成功收获 |
| **执行时间** | 200 ticks（成功）/ 1 tick（失败） |
| **注意事项** | 收获不可收获的实体会将其销毁 |

**示例：**

| **返回值** | `bool` - 是否可收获 |
| **执行时间** | 1 tick |

#### plant(entity)

| 属性 | 说明 |
|-----|------|
| **功能** | 在当前地块种植指定作物 |
| **参数** | `entity` - 作物类型（如 `Entities.Carrot`） |
| **返回值** | `bool` - 是否种植成功 |

| **注意事项** | 需要足够资源、正确的地块类型、空地 |

**示例：**
```python

    plant(Entities.Carrot)
```

#### swap(direction)

| 属性 | 说明 |
|-----|------|
| **功能** | 与指定方向的相邻作物交换位置 |
| **参数** | `direction` - 方向（North/South/East/West） |
| **返回值** | `bool` - 是否交换成功 |
| **执行时间** | 200 ticks（成功）/ 1 tick（失败） |
| **注意事项** | 不适用于所有实体，支持空地块交换 |

#### till()

| 属性 | 说明 |
|-----|------|
| **功能** | 翻耕当前地块 |
| **参数** | 无 |
| **返回值** | `None` |
| **执行时间** | 200 ticks |

#### use_item(item, n=1)

| 属性 | 说明 |
|-----|------|
| **功能** | 使用指定物品n次 |
| **参数** | `item` - 物品类型；`n` - 使用数量（默认1） |
| **返回值** | `bool` - 是否使用成功 |

| **注意事项** | 仅部分物品可用：Water、Fertilizer、Weird_Substance |

**示例：**

## 5. 开发注意事项

### 5.1 语法限制

游戏环境不是完整的Python，有以下限制：

| 不支持 | 替代方案 |
|-------|---------|
| `import` 语句 | 直接引用游戏内置对象 |
| 三引号字符串 `"""..."""` | 使用 `#` 注释 |
| 列表推导式 `[x for x in y]` | 使用普通循环构建列表 |
| `lambda` 表达式 | 使用普通函数定义 |
| `del` 关键词 | 使用 `dict.pop()` |
| 类型提示 | 省略或使用注释 |

### 5.2 性能优化

1. **减少移动次数**：规划最优路径，避免重复访问
2. **批量处理**：使用蛇形遍历减少往返
3. **能量管理**：优先收获向日葵获取能量加速
4. **水分优先级**：高价值作物优先浇水
5. **肥料使用**：在需要快速生长时使用，注意感染治愈

### 5.3 常见错误

1. **检查解锁状态**：使用 `num_unlocked()` 确认功能已解锁
2. **资源检查**：种植前确认 `num_items()` 足够
3. **地块检查**：翻耕前确认 `get_ground_type()`
4. **边界处理**：坐标超出范围会自动循环
5. **枯萎处理**：南瓜区需要及时补种枯萎南瓜

### 5.4 代码结构建议

```python
# 主循环结构示例
def main():
    while True:
        # 1. 水分管理
        water_all_zones()
        
        # 2. 高优先级区域管理
        manage_pumpkin_patch()
        
        # 3. 能量优化
        optimize_sunflower_harvest()
        
        # 4. 全面收获与补种
        harvest_and_replant_all()
        
        # 5. 伴生优化（如果已解锁）
        if num_unlocked(Unlocks.Polyculture) > 0:
            optimize_companion_planting()
```

### 5.5 调试技巧

1. 使用 `print()` 输出关键信息
2. 使用 `quick_print()` 进行无痕调试
3. 使用 `get_time()` 和 `get_tick_count()` 测量性能
4. 使用 `set_execution_speed()` 降低速度观察执行过程
5. 使用 `set_world_size()` 在小农场测试代码
